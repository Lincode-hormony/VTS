# CLAUDE.md

仓库代号 VTS（Video-To-Sheet），包名 `sam2-tool`。这份文档是给后续 agent 用的"上手地图"，目的是 5 分钟内理解项目在做什么、代码结构、数据流、dev/prod 差异，避免重复踩前面踩过的坑。

## 1. 项目在做什么

把"图 → 视频 → 序列帧 → 精灵图/ZIP"做成一个闭环。流程四步：

1. **创建角色** — 上传参考图，在统一画布里调位置 → 拿到一张固定尺寸的角色立绘
2. **图生视频** — 调火山引擎 Seedance API，把角色图当首尾帧，生成动作视频
3. **SAM2 抽帧** — 把视频送到 AutoDL 上的 SAM2 服务，交互式点选目标，拿到透明背景序列帧
4. **最终产物** — 用户选闭环帧 → 三种产物从同一份帧字节派生：
   - GIF 循环预览
   - 横向精灵图 PNG（列数可调，自带智能推荐）
   - PNG 序列帧 ZIP

**单一数据源原则**：选中的帧 → `materializeFinalFrames` 一次性物化成 `Blob[]` → GIF/Sprite/ZIP 都从这份 blob 派生。无论用户选哪种格式下载，背后是同一批像素。改这块时不要破坏这个不变式。

## 2. 技术栈

- React 19 + TypeScript + Vite 6
- Tailwind CSS 4（PostCSS 插件）
- 包管理：npm
- 部署：Cloudflare Pages（Advanced Mode 单 `_worker.js`）
- 状态持久化：localStorage（角色、动作、模板都存这里，无后端数据库）

## 4. 数据流（含三种产物的派生路径）

```
[SAM2 后端返回 frames URL[]]
         │
    用户在 FrameGallery 勾选 / 用 LoopControlPanel 自动找闭环
         │
         ▼
  materializeFinalFrames(selectedFrames)  ← 唯一源
   返回 { blobs: Blob[], width, height }
         │
   ┌─────┼─────┐
   ▼     ▼     ▼
  GIF   Sprite  ZIP
 (gifenc) (canvas) (jszip)
```

- 列数变化时**只重跑 sprite**，不重 materialize（blobs 已经在 `Sam2Workspace` state 里）
- 智能列数：`recommendSpriteColumns(N, w, h) = round(sqrt(N * h / w))`，让最终精灵图接近正方形
- GIF 因为格式限制只能 256 色 + 不透明，所以填用户选的背景色，但**输入帧和 sprite/zip 是同一批**

## 5. dev / prod 接口分层（重点，前面被这块绕了很久）

前端只调两类相对路径：

| 路径 | 用途 | dev 实现 | prod 实现 |
|---|---|---|---|
| `/api/seedance/tasks` (POST/GET) | 图生视频任务提交/查询 | `vite.config.ts` 的 `seedance-api-proxy` 中间件 | `pages-worker.mjs` |
| `/sam2/api/v1/*`（外部地址） | SAM2 服务 | 直接打 AutoDL（`VITE_SAM2_API_URL`） | 同上 |

**两边逻辑完全相同**：收到请求 → 加 `Authorization: Bearer ${ARK_API_KEY}` → 转发到 `https://ark.cn-beijing.volces.com/api/v3/contents/generations/tasks` → 把响应原样回。

**关键事实**：火山 Seedance API 直接接受 `data:image/png;base64,...` 格式的图片，**不需要先把参考图上传到任何对象存储**。仓库历史上有三套 assets 上传链路（CF KV + R2 worker + Pages functions）都已经删除。如果未来要加图片上传，请先确认是不是真的需要——base64 直传是首选。

### dev/prod 一致性

dev 通 ≠ prod 一定通，差异点：

1. **ARK_API_KEY**：dev 读 `.env.local`，prod 必须用 `wrangler pages secret put ARK_API_KEY --project-name sam2-tool` 配在 CF Pages Production 环境
2. **运行时**：dev 是 Node.js，prod 是 Cloudflare Workers V8 isolate。当前 worker 代码很轻（只是加 header + 转发），未发现兼容性问题，但加新逻辑时要意识到 prod 不能用 Node 内置模块
3. **VITE_SAM2_API_URL**：dev 在 `.env.local`，prod 在 `wrangler.toml` 的 `[vars]` 段

## 6. 常用命令

```bash
npm run dev          # vite dev server，默认 127.0.0.1:3000，端口被占自动递增
npm run build        # vite build → dist/，然后 copy-pages-worker.mjs 把 pages-worker.mjs 复制成 dist/_worker.js
npm run preview      # 本地预览生产构建
npm run typecheck    # tsc --noEmit

# 部署到 CF Pages（production 分支 = main）
npx wrangler pages deploy dist --project-name sam2-tool --branch main --commit-dirty=true

# 配 secret（首次或轮换 key）
echo 'ark-xxx' | npx wrangler pages secret put ARK_API_KEY --project-name sam2-tool

# 验证 prod
curl https://sam2-tool.pages.dev/api/seedance/tasks/health
# 期望: ok=true / hasArkKey=true / endpoint=https://ark.cn-beijing.volces.com/...
```

## 7. 外部依赖（坏了功能就挂）

| 依赖 | 用途 | 故障表现 |
|---|---|---|
| **火山引擎 Seedance**（ark.cn-beijing.volces.com） | 图生视频 | 第 2 步"生成"按钮一直 polling 或返回 failed |
| **AutoDL SAM2 服务**（`u184490-8409-...seetacloud.com:8443`） | 视频抽帧 | 第 3 步"开始提取序列帧"卡住或报 connection refused |
| **Cloudflare Pages** | 生产托管 + worker | 整个生产站不可用 |

AutoDL 实例可能因为没人开机而下线。怎么开机/启动 SAM2 服务见 `AUTODL_BACKEND_GUIDE.md`。

## 9. 关键不变式 / 容易踩的坑

- **不要重新引入 assets 上传**。火山直接吃 base64，仓库已经清理过三套重复实现，不要再加第四套
- **`frameProcessor.ts` 的 `materializeFinalFrames` 是唯一源**。新增产物（比如 webp 序列、APNG）要从 `Blob[]` 派生，不要单独再去拉一遍 SAM2 后端的图
- **localStorage 里存的是 blob URL**。blob URL 刷新页面就失效，所以 `VideoFusionWorkspace.migrateActions` 会在挂载时把旧 schema（带 `spriteSheetUrl` 等字段）的 action 清空回退到 SAM2 节点，这是故意的
- **角色 `referenceImageUrl` 是 base64 data URL**（清理 assets 之后），不是公网 URL。localStorage 单条角色约 200KB-1MB，几十个不会爆 5MB 限额
- **`_worker.js` 不要手改**。它是 `pages-worker.mjs` 的副本，build 时自动生成。改了源文件后必须 `npm run build` 才能同步到 dist
- **vite.config 改了要重启 dev server**，HMR 不接收 vite 配置变更
- **CF Pages production branch 必须是 main**。`--branch main` 落 Production，其他 branch 名会落 Preview，Preview 拿不到 production secret（`hasArkKey=false`）
- **改完 secret 要重新 deploy**，secret 变更不会自动重启已有的 worker

## 10. 给 agent 的工作风格建议

- 先读 `VideoFusionWorkspace.tsx`（顶层编排）+ `Sam2Workspace.tsx`（产物生成）+ `frameProcessor.ts`（核心算法）这三个文件就能理解 80% 的代码
- 改前端 UI 之前先跑 `npm run dev`，端口可能不是 3000（前几个被占了会往上飘）
- 改完跑 `npm run typecheck` 和 `npm run build`，两个都过才算稳
- 部署前确认 `dist/_worker.js` 和 `pages-worker.mjs` 内容一致（diff 应当无差异）
- `.codegraph/` 已索引，可以用 `codegraph_explore` 工具一次性拿到符号 + 调用链，比 grep 快
